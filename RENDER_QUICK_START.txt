ГОТОВЫЙ КОД ДЛЯ RENDER.COM

✅ Для развёртывания на Render.com всё уже подготовлено:

ФАЙЛЫ:
- Procfile ........................ Команда запуска приложения
- render.yaml .................... Конфиг для Render (опционально)
- requirements.txt ............... Все зависимости указаны
- Dockerfile ..................... Для контейнеризации
- .env.example ................... Шаблон переменных окружения

БЫСТРЫЙ СТАРТ:

1. Залить код на GitHub:
   git init
   git add .
   git commit -m "Deploy to Render"
   git push origin main

2. На Render.com создать "New Web Service"

3. Выбрать GitHub репозиторий

4. Заполнить:
   - Build Command: pip install -r requirements.txt
   - Start Command: uvicorn main:app --host 0.0.0.0 --port $PORT

5. Environment variables (добавить в Render dashboard):
   OZON_CLIENT_ID = (введёт заказчик)
   OZON_API_KEY = (введёт заказчик)
   OPENAI_API_KEY = (введёт заказчик)
   RESPONSE_TONE = friendly
   RESPONSE_SIGNATURE = Ваша подпись
   DEBUG = False

6. Нажать Deploy

Render автоматически:
- Установит зависимости
- Запустит приложение
- Выделит публичный URL
- Будет перезагружать при git push

ЦЕНЫ:
- Free: $0 (с лимитами, засыпает при неиспользовании)
- Starter: $7/месяц (гарантированный uptime)
- Standard: $12/месяц (лучше для продакшена)

Для PostgreSQL (рекомендуется для продакшена):
- Render → Databases → Create PostgreSQL
- Скопировать DATABASE_URL
- Добавить в Environment переменные
